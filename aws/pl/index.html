Virtual private Cloud (VPC) 1/2

  • wirtualna sieć w infrastrukturze AWS
  • iluzja własnego data center
  • właściciel VPC zarządza dostępem z/do sieci

Virtual private Cloud (VPC) 2/2

  • statyczna adresacja IPv4 (elastic IP)
  • wiele adresów IP wskazujących na zasób
  • definiowanie/przypisywanie interfejsów sieciowych
  • konfiguracja security group w trakcie działania zasobów VPC
  • kontrola ruchu sieciowego
  • network access control lists

EC2 Instances

Elastic Compute Cloud

EC2 Instances

  • dostarcza wirtualne maszyny
  • user wybiera typ (CPU, RAM)
  • dowolna ilość wolumenów dyskowych
  • płacisz tylko za działające maszyny

EC2 Instances - modele sprzedaży

  • on-demand
  • reserved
  • spot instances
  • on-demand - standardowa płatność za zużycie/h
  • reserved - płatnośc z wyprzedzeniem, 40% redukcji kosztów względem on-demand
  • spot - giełda nieuzywanych instancji
  • dedicated host - rezerwacja na fizyczny serwer do stawiania instancji EC2

EC2 Spot instances

  • pozwala na redukcję kosztów (nawet 10x mniejsze ceny)
  • w każdej chwili mogą zostać odebrane (2 minutes notification)

EC2 Spot Fleet

  • definiuje klaster instancji, które sa wymagane do działania
  • może zawierac mix spot-instances i on-demand
  • pozwala definiować automatyczne reguły pozwalające obniżać koszty

EC2 Instances - typy

  • General Purpose
  • Compute optimized
  • Accelerate Computing (GPU)
  • Storage optimized

Amazon Machine Image (AMI)

  • zawiera pełen obraz instancji
  • obrazy mogą być płatne (np. AWS Linux)
  • AWS nie pobiera opłat za obrazy systemów open-source (np. Ubuntu)
  • możliwośc przechowywania własnych AMI

Instance store

  • storage tymczasowy
  • dane sa ulotne wskutek usunięcia maszyny lub błędu dysków
  • dobry na bufory, cache
maszyny posiadające zasoby instance-store nie mają funkcji STOP

Elastic Block Storage (EBS)

  • block storage dla instancji
  • wysoka dostępność i solidność
  • rekomendowany dla szybkiego dostępu do danych
  • rekomendowany dla długiego utrzymania danych
  • wspiera szyfrowanie

Security groups

  • firewall działający na warstwie 4
  • filtruje pakiety ICMP
  • filtruje pakiety TCP/UDP
  • umożliwia filtrowanie po zakresie portów
  • umożliwia filtrowanie po zakresie adresów IP
  • działa na połączenia przychodzące i wychodzące

Elastic IP

  • adres IPv4 powiązany z kontem AWS
  • można go dowolnie przepinać
  • brak wsparcia dla IPv6
  • płatny, gdy nieużywany

Network interfaces

  • reprezentuje wirtualną kartą sieciową
  • wiele prywatnych adresów IPv4 z puli VPC
  • jeden publiczny IPv4
  • wiele publicznych IPv6
  • MAC adres
  • wiele security group
Liczba interfejsów możliwych do przypisania na instancję zależy od jej typu. Liczba adresów IP możliwych do przypisania równiez się różni w zalezności od typu instancji.
