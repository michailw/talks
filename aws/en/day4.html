Relational Database Service (RDS)

RDS

  • database as a service
  • allows to scale CPU+memory, storage, IOPS independently
  • backups, fail detection, data recovery
  • db engine autoupdates
  • simplified data replication

RDS - db engines

  • Aurora / MySQL / MariaDB
  • PostgreSQL
  • MS SQL
  • Oracle

RDS - limits

  • instance type: magnetic, gpo, iops
  • no access to system schemas
  • no access through SSH
Types: Magnetic, General Purpose (SSD), Provisioned IOPS

RDS - backups

  • manual
  • automated (scheduled)
  • first backup is a full snapshot of DB
  • each next backup is incremental

CloudFront

CloudFront 1/2

  • content delivery network (CDN)
  • high transfer rate
  • low latency
  • close to end users
  • 155 Edge Locations
  • 11 Regional Edge Caches
  • 166 Points of Presence
  • direct connection to AWS network

CloudFront 2/2

  • supports only HTTP
  • supports websockets
  • backend sources: S3, EC2, LB, Lambda
  • Lambda@Edge
  • detects device type (desktop/mobile)
  • detects country of origin
  • full controll of cache behaviour
